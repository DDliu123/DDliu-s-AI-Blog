## 2. 文档概述

本文档描述一款 AI 日记软件的产品需求。该软件除了支持用户记录日记外，还能让 AI 分析日记内容，生成每日核心记录，并将关键内容沉淀为 AI 记忆。系统每天早上 8:00 自动随机输出《易经》64 卦中的一个卦象，并结合用户的日记记忆，生成今日运势分析。

本需求重点覆盖日记记录、AI 总结、AI 记忆、每日卦象、运势生成、隐私安全、自动任务和内容边界。

## 3. 背景与问题

## 3.1 背景

很多用户有记录日记、复盘生活、整理情绪和规划未来的需求。但传统日记软件通常只提供记录、搜索、标签和时间线能力，用户写下的内容很少被进一步利用。

随着 AI 能力发展，日记内容可以从“静态记录”变成“可被理解和持续引用的个人记忆”。AI 可以帮助用户从日记中提取重要事件、情绪变化、长期目标、人际关系和生活主题，并在未来生成更贴近用户状态的反馈。

同时，部分用户对每日仪式感、运势、占卜、易经和自我反思内容感兴趣。如果将 64 卦与个人日记记忆结合，可以形成一种兼具记录、陪伴、反思和每日启发的产品体验。

## 3.2 当前问题

| 问题 | 说明 |
| --- | --- |
| 日记内容利用率低 | 用户写完日记后，内容通常只是被存档，很少被重新组织和利用。 |
| 用户难以坚持复盘 | 每天手动总结重要内容有负担，容易中断。 |
| AI 缺少个人上下文 | 普通 AI 工具不了解用户长期经历，给出的建议较泛。 |
| 运势内容容易空泛 | 传统每日运势通常不结合用户真实生活，缺少个人相关性。 |
| 个人记录隐私敏感 | 日记包含情绪、关系、工作、健康等私密信息，需要明确保护机制。 |

## 3.3 为什么现在做

现在 AI 已经能够对长文本进行总结、分类、抽取和个性化生成。用户也逐渐接受 AI 作为个人助理、情绪陪伴和自我复盘工具。因此，可以把日记、AI 记忆和每日运势结合起来，形成一个更具个人化和持续性的记录产品。

## 4. 产品目标与成功指标

## 4.1 产品目标

本产品希望帮助用户完成三件事：

1. 低成本记录每日生活、情绪、想法和事件。
2. 自动提取每日重要内容，形成可回顾的核心记录。
3. 基于用户长期记忆和每日随机卦象，生成有启发感的今日运势分析。

## 4.2 用户价值

| 用户价值 | 说明 |
| --- | --- |
| 更容易坚持记录 | 用户只需要写日记，AI 自动辅助整理重点。 |

| 获得每日仪式感 | 每天早上 8:00 收到一个结合自身状态的卦象分析。 |
| 获得个性化建议 | 运势分析不只基于通用卦象，还结合用户真实记录。 |
| 形成个人 AI 记忆 | 用户的经历、偏好、目标和重要关系可被持续引用。 |

## 4.3 业务价值

| 业务价值 | 说明 |
| --- | --- |
| 提升用户留存 | 每日 8:00 自动内容可形成固定打开习惯。 |

## 6. 需求范围

## 6.1 MVP 本期做什么

- 支持用户创建、编辑、删除、查看日记。
- 支持按日期查看日记列表。
- 支持 AI 根据当天日记生成每日核心记录。
- 支持 AI 从每日核心记录中提取候选记忆。
- 支持用户确认、编辑或拒绝候选记忆。
- 支持系统每天早上 8:00 自动随机抽取 64 卦中的一个卦象。
- 支持 AI 结合卦象、卦义和用户记忆生成今日运势分析。
- 支持查看历史每日卦象和运势。
- 支持基础隐私设置和数据删除。

## 6.2 MVP 本期不做什么

- 不做社交分享社区。
- 不做多人协作日记。
- 不做专业心理咨询或医疗诊断。
- 不做真实投资、疾病、法律等高风险决策建议。
- 不做复杂的周易起卦规则，如铜钱起卦、梅花易数、六爻纳甲等。
- 不做完整的多端同步冲突解决。
- 不做语音日记和图片日记，除非后续版本扩展。

## 6.3 后续版本可能做什么

- 支持语音输入和语音转日记。
- 支持图片、位置、天气等多模态记录。
- 支持周报、月报、年度人生报告。
- 支持长期情绪趋势分析。
- 支持用户自定义每日运势风格。
- 支持不同占卜体系，如塔罗、星座、生肖等。
- 支持本地优先存储和端到端加密。
- 支持个人知识库和 AI 对话。

## 7. 核心用户流程

## 7.1 写日记与生成核心记录流程

```markdown
1. 用户进入日记首页。
2. 用户点击「写日记」。
3. 用户输入标题、正文和可选标签。
4. 用户保存日记。
5. 系统保存原始日记。
6. 用户点击「生成今日核心记录」。
7. AI 读取当日所有日记内容。
8. AI 输出今日核心记录。
9. 用户可保存、重新生成或手动编辑核心记录。
```

## 7.2 AI 记忆写入流程

```markdown
1. AI 完成今日核心记录后，系统提取候选记忆。
2. 候选记忆按类型展示给用户。
3. 用户逐条确认、编辑或拒绝。
4. 用户确认后，记忆写入 AI 记忆库。
5. 后续每日运势生成时，可引用这些记忆。
```

## 7.3 每日 8:00 卦象运势流程

```markdown
1. 每天早上 8:00，系统启动每日运势任务。
2. 系统检查用户是否开启每日运势。
3. 系统随机抽取 64 卦中的一个卦象。
4. 系统读取该卦象的基础解释。
5. 系统读取用户已确认的 AI 记忆。
6. AI 结合卦象、日期和用户记忆生成今日运势。
7. 系统保存今日卦象和运势结果。
8. 若用户开启通知，系统发送提醒。
9. 用户打开 App 查看今日运势详情。
```

## 8. 功能需求

## 8.1 日记记录

### 功能说明

用户可以按日期记录日记。日记是所有 AI 分析和记忆生成的基础数据。

### 页面入口

- 首页「写日记」按钮
- 日历页某一天
- 今日运势页中的「记录今天」入口

### 字段说明

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 日期 | 日期 | 是 | 默认当天，可手动调整 |
| 标题 | 文本 | 否 | 用户可自定义标题 |
| 正文 | 长文本 | 是 | 日记正文 |
| 标签 | 多选 / 自定义 | 否 | 如工作、关系、健康、情绪 |
| 心情 | 枚举 | 否 | 如开心、平静、焦虑、低落 |
| 创建时间 | 时间 | 是 | 系统生成 |
| 更新时间 | 时间 | 是 | 系统生成 |

### 业务规则

- 同一天可以写多篇日记。
- 日记正文为空时，不允许保存。
- 用户可编辑和删除自己的日记。
- 删除日记后，不自动删除已经确认写入的 AI 记忆，需用户另行管理。
- 若用户删除当日日记，系统应提示其可能影响当日核心记录质量。

### 验收标准

| 场景 | 前置条件 | 操作 | 预期结果 |
| --- | --- | --- | --- |
| 创建日记 | 用户已登录 | 输入正文并保存 | 日记保存成功 |
| 编辑日记 | 日记存在 | 修改内容并保存 | 日记内容更新 |
| 删除日记 | 日记存在 | 点击删除并确认 | 日记被删除 |
| 空正文保存 | 正文为空 | 点击保存 | 系统提示正文不能为空 |

## 8.2 AI 每日核心记录

### 功能说明

AI 根据用户当天的日记内容，生成一份结构化的每日核心记录。该记录用于帮助用户快速回顾一天，也作为候选记忆提取的来源。

### 输出结构

| 模块 | 说明 |
| --- | --- |
| 今日摘要 | 用 3 到 5 句话概括今天最重要的内容 |
| 关键事件 | 提取 1 到 5 个重要事件 |
| 情绪状态 | 总结当天主要情绪和变化 |
| 重要关系 | 提取与他人相关的重要互动 |
| 今日收获 | 总结用户的反思、学习或进展 |
| 待关注事项 | 提醒后续可能需要跟进的问题 |

### 业务规则

- 只有当日存在至少一篇日记时，才能生成核心记录。
- 用户可以重新生成核心记录。
- 用户可以手动编辑 AI 生成内容。
- 系统需保留最近一次保存版本。
- AI 不应编造日记中没有出现的事实。
- 如果日记内容较少，AI 应明确说明“基于当前记录较少，以下总结可能不完整”。

### AI 生成要求

- 语言应温和、清晰、克制。
- 不进行医学、法律、投资等专业判断。
- 不给用户贴负面人格标签。
- 不把短期情绪直接判断为长期问题。
- 对不确定内容使用“可能”“看起来”“从记录中能看到”等表达。

### 验收标准

| 场景 | 前置条件 | 操作 | 预期结果 |
| --- | --- | --- | --- |
| 生成核心记录 | 当天有日记 | 点击生成 | AI 输出结构化核心记录 |
| 无日记生成 | 当天无日记 | 点击生成 | 系统提示需要先写日记 |
| 重新生成 | 已有核心记录 | 点击重新生成 | 系统生成新版本并提示用户确认 |
| 编辑记录 | 已有核心记录 | 用户修改并保存 | 修改后的内容被保存 |

## 8.3 AI 记忆

### 功能说明

AI 从每日核心记录中提取对未来有长期参考价值的信息，生成候选记忆。用户确认后，这些内容会写入 AI 记忆库，用于后续运势分析和个性化反馈。

### 记忆类型

| 类型 | 示例 |
| --- | --- |
| 长期目标 | 用户正在准备转行、考证、创业等 |
| 重要关系 | 用户经常提到某位家人、伴侣、朋友或同事 |
| 偏好 | 用户喜欢早起、写作、独处、运动等 |
| 压力来源 | 用户近期工作压力来自项目延期 |
| 重复主题 | 用户多次提到自我怀疑、时间管理、亲密关系等 |
| 重要事件 | 用户开始新工作、搬家、分手、结婚等 |

### 记忆字段

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 记忆内容 | 文本 | 是 | 一条清晰、简短的事实或偏好 |
| 记忆类型 | 枚举 | 是 | 目标、关系、偏好、压力、事件等 |
| 来源日期 | 日期 | 是 | 来自哪一天的核心记录 |
| 来源日记 | 引用 | 否 | 可追溯到原始日记 |
| 置信度 | 数值 | 否 | AI 对该记忆准确性的判断 |
| 状态 | 枚举 | 是 | 待确认、已确认、已拒绝、已删除 |
| 创建时间 | 时间 | 是 | 系统生成 |
| 更新时间 | 时间 | 是 | 系统生成 |

### 业务规则

- AI 只能生成候选记忆，不能默认写入长期记忆。
- 用户必须确认后，记忆才可用于后续生成。
- 用户可以编辑、删除、停用记忆。
- 用户删除记忆后，后续 AI 不再主动引用该记忆。
- 对敏感信息，如健康、财务、亲密关系，默认需更谨慎展示。
- 记忆应尽量写成中性事实，不应写成武断评价。

### 候选记忆示例

```markdown
- 用户近期在关注职业转型，并多次提到希望建立自己的产品作品集。
- 用户在工作中容易受到临时需求打断，影响专注感。
- 用户重视早晨的仪式感，希望通过固定节奏开始一天。
```

### 验收标准

| 场景 | 前置条件 | 操作 | 预期结果 |
| --- | --- | --- | --- |
| 生成候选记忆 | 已有核心记录 | 点击提取记忆 | 系统展示候选记忆 |
| 确认记忆 | 存在候选记忆 | 用户点击确认 | 记忆写入 AI 记忆库 |
| 编辑记忆 | 存在候选记忆 | 用户修改后确认 | 修改后的记忆被保存 |
| 拒绝记忆 | 存在候选记忆 | 用户点击拒绝 | 该记忆不写入记忆库 |
| 删除记忆 | 已确认记忆存在 | 用户点击删除 | 后续 AI 不再引用该记忆 |

## 8.4 64 卦数据

### 功能说明

系统内置《易经》64 卦基础数据。每日运势生成时，系统从 64 卦中随机抽取一个卦象。

### 卦象字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| 卦序 | 数字 | 1 到 64 |
| 卦名 | 文本 | 如乾为天、坤为地 |
| 卦象 | 文本 | 上卦和下卦组合 |
| 卦辞简释 | 文本 | 用现代语言解释基础含义 |
| 关键词 | 数组 | 如开始、积累、等待、转折 |
| 今日启发 | 文本 | 适合每日运势使用的解释 |

### 随机规则

- 每个用户每天只生成一个卦象。
- 同一用户同一天多次打开，展示同一个结果。
- 每天 8:00 后首次生成的结果保存到历史记录。
- 若 8:00 自动任务失败，用户当天首次打开 App 时补偿生成。
- MVP 阶段使用纯随机抽取，不引入复杂起卦算法。

## 8.5 每日卦象与今日运势

### 功能说明

系统每天早上 8:00 自动为用户生成今日卦象和运势分析。内容由“随机卦象 + 卦义解释 + 用户 AI 记忆 + 最近日记摘要”共同生成。

### 内容结构

| 模块 | 说明 |
| --- | --- |
| 今日卦象 | 展示卦名、卦序、卦象关键词 |
| 卦象简释 | 用现代语言解释该卦含义 |
| 今日总运 | 结合用户近况给出整体分析 |
| 工作 / 学习 | 对工作、学习、项目推进给出提醒 |
| 情绪 / 关系 | 对情绪状态、人际互动给出建议 |
| 今日宜做 | 给出 1 到 3 个适合今天做的行动 |
| 今日慎做 | 给出 1 到 3 个需要避免的行为 |
| 给自己的提醒 | 用一句话形成今日提醒 |

### 生成规则

- 运势内容必须基于卦义和用户已确认记忆。
- 可以引用最近 7 天核心记录，但不能暴露过多原始隐私细节。
- 如果用户没有足够记忆，系统生成更通用的卦象解读。
- 内容应定位为“反思与启发”，不能宣称可以预测未来。
- 不应输出恐吓式、宿命式或绝对化表达。
- 不应给出医疗、法律、投资等高风险建议。

### 示例输出

```markdown
今日卦象：第 3 卦，水雷屯

关键词：开始、混乱、积累、耐心

今日总运：
今天更适合打基础，而不是急着证明结果。结合你近期反复提到的职业转型和作品集计划，今天的重点不是做出一个完整成果，而是把一个具体模块推进到可展示的程度。

工作 / 学习：
适合整理需求、拆分任务、补齐资料。不要同时开启太多新方向。

情绪 / 关系：
如果感到焦虑，可以先把问题写下来，再决定是否需要回应别人。今天不适合在情绪高点做重要沟通。

今日宜做：
- 整理一个明确的小任务。
- 复盘最近 3 天反复出现的问题。
- 给重要项目留出一段不被打扰的时间。

今日慎做：
- 不要因为进展慢就否定整个方向。
- 不要在信息不足时做长期承诺。

给自己的提醒：
先稳住节奏，新的秩序会从混乱中长出来。
```

### 验收标准

| 场景 | 前置条件 | 操作 | 预期结果 |
| --- | --- | --- | --- |
| 自动生成运势 | 用户开启每日运势 | 到达 8:00 | 系统生成今日卦象和运势 |
| 重复打开 | 今日已生成 | 用户再次打开 | 展示同一份结果 |
| 自动任务失败 | 8:00 任务失败 | 用户当天首次打开 App | 系统补偿生成今日运势 |
| 无记忆用户 | 用户没有确认记忆 | 生成运势 | 输出通用但不空泛的卦象解读 |
| 关闭运势 | 用户关闭每日运势 | 到达 8:00 | 系统不生成、不通知 |

## 8.6 历史记录

### 功能说明

用户可以按日期查看过去的日记、核心记录、AI 记忆和每日卦象运势。

### 页面内容

| 内容 | 说明 |
| --- | --- |
| 原始日记 | 当天用户写下的日记 |
| 核心记录 | AI 总结后的每日重点 |
| 写入记忆 | 当天确认写入的 AI 记忆 |
| 今日卦象 | 当天抽取的卦象 |
| 运势分析 | 当天生成的完整运势 |

### 业务规则

- 用户只能查看自己的历史记录。
- 删除日记时，历史卦象和运势默认保留。
- 用户可删除某一天的运势记录。
- 若用户删除所有相关日记和记忆，历史运势不应继续用于未来生成。

## 8.7 设置与隐私

### 功能说明

用户可以管理 AI 记忆、每日运势开关、通知时间和数据删除。

### 设置项

| 设置项 | 默认值 | 说明 |
| --- | --- | --- |
| AI 总结 | 开启 | 是否允许 AI 分析日记生成核心记录 |
| AI 记忆 | 手动确认 | 候选记忆必须经用户确认 |
| 每日运势 | 开启 | 是否每天生成卦象运势 |
| 通知时间 | 08:00 | MVP 固定为 8:00，后续可自定义 |
| 隐私模式 | 关闭 | 开启后隐藏首页敏感内容 |
| 删除全部数据 | 无 | 用户可申请删除日记、记忆和运势数据 |

### 隐私规则

- 日记内容属于高度敏感数据。
- AI 记忆写入必须由用户确认。
- 用户可以查看 AI 记忆来源。
- 用户可以随时删除日记、核心记录、记忆和运势。
- 删除后的内容不应继续用于后续 AI 生成。
- 产品应清楚告知用户哪些数据会被 AI 使用。

## 11. 权限、安全与合规

## 11.1 权限要求

- 用户必须登录后才能写日记和生成 AI 记忆。
- 用户只能访问自己的日记、记忆和运势。
- 管理员不应在后台直接查看用户原始日记，除非有明确授权和审计机制。

## 11.2 数据安全

- 日记正文、AI 记忆和运势记录应加密存储。
- 所有数据传输应使用 HTTPS。
- 敏感操作，如删除全部数据，应进行二次确认。
- 应记录关键数据操作日志。

## 11.3 AI 使用告知

产品需要明确告知用户：

- AI 会读取用户日记来生成核心记录。
- AI 只会在用户确认后写入长期记忆。
- 每日运势会使用已确认记忆和最近核心记录。
- 运势内容仅供反思和娱乐，不代表确定预测。

## 11.4 内容安全边界

AI 不应输出以下内容：

- 恐吓用户的宿命式判断。
- 医疗诊断、用药建议或心理疾病诊断。
- 投资买卖建议。
- 法律结论。
- 鼓励自伤、伤害他人或违法行为的内容。
- 基于用户隐私做羞辱、歧视或道德审判。

## 12. 异常与边界情况

| 场景 | 系统处理 | 用户提示 |
| --- | --- | --- |
| AI 总结失败 | 允许重试，并记录失败原因 | 生成失败，请稍后重试 |
| 8:00 自动任务失败 | 当天首次打开 App 时补偿生成 | 今日运势正在生成 |
| 用户无日记 | 不生成核心记录 | 今天还没有日记，先记录一点吧 |
| 用户无 AI 记忆 | 生成通用卦象运势 | 当前运势基于卦象生成，记录越多会越贴近你 |
| AI 生成内容过长 | 自动截断或要求重新生成 | 内容较长，已为你整理重点 |
| 用户关闭 AI 记忆 | 不生成候选记忆 | AI 记忆已关闭 |
| 用户删除全部数据 | 停止使用历史数据生成 | 数据删除后无法恢复 |
| 同一天重复触发任务 | 返回已生成结果 | 无需提示 |

## 13. 非功能需求

## 13.1 性能

- 日记列表加载时间应小于 2 秒。
- 今日运势页已有结果时加载时间应小于 2 秒。
- AI 核心记录生成时间目标小于 20 秒。
- 每日运势生成时间目标小于 20 秒。

## 13.2 稳定性

- 自动任务需要支持失败重试。
- 每日卦象生成必须幂等，同一天不能重复生成多个结果。
- AI 生成失败不能影响用户查看原始日记。
- 系统需要保存生成状态，避免重复扣费或重复调用模型。

## 13.3 兼容性

- MVP 优先支持 Web 或移动端单端体验。
- 后续多端同步时，需要处理离线写入和冲突合并。

## 13.4 可维护性

- 64 卦基础数据应结构化存储，便于后续编辑。
- AI Prompt 应版本化管理。
- 每次 AI 生成结果应记录 Prompt 版本和模型版本。
- 记忆抽取规则应支持后续调整。

## 14. AI Prompt 设计要求

## 14.1 每日核心记录 Prompt 要求

输入：

- 当天所有日记正文
- 用户可选心情和标签

输出：

- 今日摘要
- 关键事件
- 情绪状态
- 重要关系
- 今日收获
- 待关注事项

约束：

- 不编造事实。
- 不做诊断。
- 不进行道德审判。
- 不把临时情绪过度概括为人格特征。

## 14.2 AI 记忆抽取 Prompt 要求

输入：

- 今日核心记录
- 已有 AI 记忆列表

输出：

- 候选记忆列表
- 记忆类型
- 置信度
- 推荐理由

约束：

- 避免重复记忆。
- 只抽取对未来有长期参考价值的信息。
- 敏感内容要保守处理。
- 必须等待用户确认后写入。

## 14.3 今日运势 Prompt 要求

输入：

- 今日日期
- 今日卦象
- 卦象解释
- 用户已确认 AI 记忆
- 最近 7 天核心记录摘要

输出：

- 今日卦象
- 卦象简释
- 今日总运
- 工作 / 学习
- 情绪 / 关系
- 今日宜做
- 今日慎做
- 给自己的提醒

约束：

- 明确定位为启发和反思。
- 不做确定性预测。
- 不输出高风险建议。
- 不泄露过多用户原始日记细节。
- 语言应温和、具体、可行动。

## 16. 上线方案

## 16.1 灰度策略

- 第一阶段：内部测试，验证 AI 生成质量和隐私边界。
- 第二阶段：邀请 50 到 100 名种子用户使用。
- 第三阶段：开放注册，观察留存和反馈。

## 16.2 回滚方案

- 若 AI 生成质量不稳定，可临时关闭 AI 总结和运势生成。
- 若每日任务异常，可暂停自动推送，保留手动生成入口。
- 若出现隐私风险，应立即停止相关 AI 调用并排查数据链路。

## 16.3 运营准备

- 准备产品说明页，解释 AI 记忆和每日运势机制。
- 准备隐私说明，明确用户数据如何被使用。
- 准备用户反馈入口，收集运势是否准确、有帮助、冒犯或空泛。

## 16.4 客服准备

客服需要能回答：

- AI 为什么会知道这些内容？
- 如何删除 AI 记忆？
- 如何关闭每日运势？
- 删除日记后，AI 是否还会继续引用？
- 运势内容是否代表真实预测？

## 17. 风险与假设

## 17.1 核心假设

| 假设 | 验证方式 |
| --- | --- |
| 用户愿意让 AI 总结自己的日记 | 观察 AI 总结使用率 |
| 用户愿意确认 AI 记忆 | 观察记忆确认率 |
| 每日 8:00 运势能提升打开率 | 观察每日运势打开率 |
| 64 卦结合日记能提供差异化体验 | 收集用户满意度和访谈反馈 |
| 用户能理解运势是反思工具，不是确定预测 | 观察内容投诉和用户反馈 |

## 17.2 产品风险

| 风险 | 影响 | 应对方式 |
| --- | --- | --- |
| AI 总结不准确 | 用户不信任产品 | 提供编辑、重生成和反馈入口 |
| AI 记忆冒犯用户 | 用户产生隐私和心理不适 | 记忆必须手动确认，敏感内容保守处理 |
| 运势内容过度玄学 | 影响产品可信度 | 强调启发和反思，避免绝对预测 |
| 日记数据泄露 | 严重信任危机 | 加密存储、访问控制、审计日志 |
| 用户过度依赖运势 | 影响决策质量 | 明确免责声明，避免高风险建议 |
| 自动任务成本过高 | 影响商业模型 | 对活跃用户生成，限制重复调用 |

## 20. 结论

这个产品的核心不是“日记 + 占卜”的简单组合，而是把个人记录转化为可持续使用的 AI 记忆，再用每日卦象作为仪式化入口，帮助用户理解自己的状态、看见近期主题，并获得具体的今日行动提醒。

MVP 应优先验证三件事：用户是否愿意持续写日记，是否愿意确认 AI 记忆，以及每日 8:00 的卦象运势是否能形成稳定打开习惯。在验证成立之前，应控制功能范围，重点打磨 AI 总结质量、记忆准确性和隐私信任。
