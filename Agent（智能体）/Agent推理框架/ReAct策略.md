# ReAct框架：让AI Agent像人一样思考与行动——从原理到自动定价系统实现

在大模型应用开发中，如何让AI Agent摆脱“要么空想、要么盲动”的困境，真正像人类一样兼具思考深度与行动能力？LangChain中的ReAct框架给出了答案。本文将从ReAct框架的核心逻辑出发，结合鲜花自动定价的实操案例，带您全面理解这一框架的价值与落地方法。


## 一、从花店需求出发：为什么需要ReAct框架？

在一家注重用户体验的花店中，UI设计师小雪不仅专注于购花界面的美观，更希望通过节日知识、花语解读、色彩心理学等细节，让整个购花过程充满仪式感。而程序员咖哥则带领团队搭建了两大核心系统：一是能理解用户需求、结合购买历史的AI商品推荐模型，二是可预测市场需求、优化库存的动态管理系统。

  

一次沟通中，小雪提出了新的需求：“能不能让AI根据天气（比如雨天影响配送效率）、交通状况（比如高峰期配送成本上升）自动调整鲜花价格？”这个需求看似简单，却指向一个核心问题——传统AI要么只擅长“空想”（纯推理，易脱离实际），要么只擅长“盲动”（纯执行，缺乏规划），很难像人类一样“先琢磨、再行动、再调整”。此时，ReAct框架成为了实现这一需求的关键工具。

  

  

## 二、ReAct框架：模拟人类解决问题的核心逻辑

### 1. 本质：“观察-思考-行动”的循环迭代

ReAct框架的精髓，在于复刻了人类解决问题的自然流程。以小雪给鲜花定价的过程为例：她会先“思考”定价的核心要素（成本、市场需求、用户心理），再“观察”成本价、竞品价格等信息，最后“行动”（确定加价幅度、定下最终售价）。这一过程中，“思考”与“行动”并非单向进行，而是会根据实际情况反复迭代——比如发现市场价格波动后，会重新思考加价策略，再次调整定价。

  

在AI Agent中，ReAct框架将这一过程规范化：

- **推理（Reasoning）**：对应“思考+观察”，Agent会分析任务目标（如“定价”），判断现有信息是否足够，明确下一步需要补充的内容（如“需要查询当前玫瑰花市场单价”）。
    
- **行动（Action）**：对应“执行操作”，Agent根据推理结果调用外部工具（如搜索引擎查价格、计算器算加价），获取新信息后再次进入推理环节，直到完成任务。
    

### 2. 优势：弥补纯推理与纯执行的缺陷

在ReAct框架出现前，AI解决问题的方式存在明显短板：

- **纯推理（如思维链）**：仅依赖模型内部知识，容易产生“幻觉”（编造不存在的信息），且一旦初始判断错误，后续推理会像“滚雪球”一样累积错误。例如直接问模型“苹果遥控器最初为哪种设备设计”，模型可能仅凭记忆错答“Apple TV”。
    
- **纯执行（如SayCan、Web** **GPT****）**：仅擅长调用工具执行指令，却缺乏全局规划，像“没头苍蝇”一样盲目操作，无法根据结果调整策略。
    

而ReAct框架通过“推理-行动”的闭环，完美规避了这些问题：

- **结果更准确**：主动调用外部工具（如搜索引擎）获取实时、可靠的信息，避免被模型内部过时知识误导。
    
- **过程可解释**：每一步“思考（为什么调用工具）”“行动（调用了什么工具）”“结果（工具返回了什么）”都清晰可追溯，便于调试和信任。
    
- **策略可迭代**：根据工具返回的新信息动态调整思路，比如查完玫瑰花单价后，发现价格高于预期，会重新计算加价5%后的合理性。
    

### 3. 实例验证：ReAct如何解决“刁钻问题”

以“除了苹果电视，苹果遥控器最初为哪种设备设计”这一非常识问题为例，不同方法的表现差异显著：

- **零样本推理**：直接输出答案，正确率低（如错答“Apple TV”）。
    
- **思维链**：分步思考但仍依赖内部知识，最终仍可能出错。
    
- **ReAct框架**：
    
    - **推理**：“这个问题我不确定答案，需要搜索苹果遥控器的历史设计背景。”
        
    - **行动**：调用搜索引擎，搜索“苹果遥控器 最初设计用途”。
        
    - **观察**：获取信息“苹果遥控器最初为Apple Macintosh电脑设计，用于远程控制媒体播放”。
        
    - **推理**：“搜索结果明确，无需进一步补充信息，可以确定答案。”
        
    - **行动**：输出最终答案“Apple Macintosh”。
        

通过这一过程，ReAct不仅得出了正确答案，更让整个思考路径一目了然。

  

  

## 三、ReAct框架落地：在LangChain中构建自动定价Agent

要将ReAct框架从理论转化为能干活的AI Agent，需依托LangChain工具链，聚焦四大核心要素，并通过实操完成鲜花自动定价任务。

### 1. 四大核心要素：搭建Agent的“基础设施”

在LangChain中，一个完整的ReAct Agent由四部分组成，缺一不可：

  

|   |   |   |
|---|---|---|
|核心要素|作用|实例|
|**大模型**|Agent的“大脑”，负责推理（分析任务、判断下一步行动）和生成最终答案|OpenAI GPT系列、 Claude等|
|**提示词（Prompt）**|指导大模型“如何思考”，定义ReAct的行为模式|遵循“Thought-Action-Action Input-Observation”格式的ReAct模板|
|**外部工具**|Agent的“手脚和眼睛”，执行具体操作（获取外部信息、计算等）|搜索引擎（SerpAPI）、计算器（Lambeth）、数据库查询工具|
|**Agent执行器**|Agent的“指挥官”，协调大模型与工具的交互，控制任务流程（调用工具→获取结果→再次推理→直至输出答案）|LangChain的Agent Executor|

  

其中，**外部工具**又分为两类：

- **单个工具**：独立的“技能包”，如SerpAPI（用于Google搜索）、Lambeth（用于数学计算），需明确告诉Agent工具的用途和使用方法。
    
- **工具包（Toolkits）**：整合相关工具的“工具箱”，如Gmail Toolkit（包含连接邮箱、读邮件、发邮件等功能），便于一次性调用多个关联工具。
    

### 2. 实操：实现玫瑰花自动定价系统

#### 任务目标

让Agent查询当前市场玫瑰花的进货单价，在此基础上加价5%，输出最终定价。

  

#### 具体步骤

##### （1）环境准备

- **安装依赖**：安装Python包`numax Pro`（支撑Lambeth计算器工具运行）。
    
- **获取API密钥**：准备OpenAI API密钥（驱动大模型）和SerpAPI密钥（调用Google搜索）。
    

##### （2）初始化核心组件

1. **加载大模型**：选用OpenAI模型，将`temperature`设为0（减少随机性，保证推理和计算的准确性）。
    
2. **加载外部工具**：
    
    1. 工具1：SerpAPI（用于搜索“当前玫瑰花市场进货单价”）。
        
    2. 工具2：Leo Math（用于计算“单价+5%加价”后的最终价格）。
        
3. **设计ReAct提示模板**：这是控制Agent行为的关键，模板需明确要求Agent遵循固定格式输出：
    
    ```Plain
    请根据以下步骤解决问题：
    1. Thought（思考）：分析当前任务，判断是否需要调用工具，需要调用什么工具。
    2. Action（行动）：指定要调用的工具（如“SerpAPI”“Leo Math”）。
    3. Action Input（行动输入）：给工具的具体指令（如“搜索2024年玫瑰花市场进货单价”）。
    4. Observation（观察）：记录工具返回的结果。
    重复以上步骤，直到可以给出最终答案，用“Final Answer”开头输出。
    ```
    

##### （3）创建并运行Agent

4. **创建Agent**：使用LangChain的`create_react_agent`函数，将大模型、工具、提示模板整合，生成具备ReAct能力的Agent实例。
    
5. **创建执行器**：通过`Agent Executor`绑定Agent与工具，设置`verbose=True`（输出详细日志，便于观察思考过程）。
    
6. **执行任务**：向Agent输入问题“目前市场上玫瑰花的一般进货价格是多少？在此基础上加价5%应如何定价？”，Agent会自动执行以下流程：
    
    1. Thought：“我需要先知道玫瑰花的市场进货单价，才能计算加价5%后的价格，因此需要调用SerpAPI搜索。”
        
    2. Action：SerpAPI
        
    3. Action Input：“2024年玫瑰花市场进货单价 元/支”
        
    4. Observation：“搜索结果显示，2024年玫瑰花市场进货单价约为3元/支”
        
    5. Thought：“已获取单价，下一步需用计算器计算3元加5%后的价格，调用Leo Math工具。”
        
    6. Action：Leo Math
        
    7. Action Input：“3 * (1 + 5%)”
        
    8. Observation：“计算结果为3.15”
        
    9. Thought：“已获取最终定价，无需进一步操作。”
        
    10. Final Answer：“目前市场上玫瑰花的一般进货价格约为3元/支，加价5%后的定价为3.15元/支。”
        

## 四、总结：ReAct框架的价值与应用场景

ReAct框架的核心贡献，在于让AI Agent从“被动执行指令”升级为“主动解决问题”。它不局限于某一特定领域，而是适用于所有需要“结合实时信息、动态调整策略”的任务，除了自动定价，还可应用于：

- 智能客服：根据用户问题调用知识库、订单系统，逐步排查并解决问题（如“查询物流→解释延迟原因→提供补偿方案”）。
    
- 数据分析：调用数据库工具获取数据、调用可视化工具生成图表、结合业务逻辑输出分析报告。
    
- 智能家居控制：结合天气工具（如“是否下雨”）、用户位置工具（如“是否到家”），自动调整灯光、空调模式。
    

对于开发者而言，掌握ReAct框架的关键，在于理解“推理与行动协同”的本质——与其让模型“闭门造车”，不如赋予它“思考的能力”和“动手的工具”，让AI真正像人类一样，在实践中迭代优化，高效解决复杂问题。
