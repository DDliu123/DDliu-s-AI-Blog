## 主流 Agent 范式

1. ### ReAct（Reasoning + Acting）
    

交替进行**推理**和**行动**，每步先思考再执行工具调用，根据观察结果继续推理。是目前最广泛使用的范式。

2. ### Plan-and-Execute（规划执行分离）
    

先由 Planner 制定完整计划，再由 Executor 逐步执行。适合任务复杂、步骤可预见的场景。代表：LangChain Plan-and-Execute。

3. ### ReWOO（Reasoning WithOut Observation）
    

在执行前一次性规划所有工具调用，避免中间等待，减少 LLM 调用次数，提升效率。

4. ### Reflexion（反思自我改进）
    

Agent 执行后对结果进行**自我评估和反思**，将经验存入记忆，在下次尝试中改进。适合需要迭代优化的任务。

5. ### Tool-Use / Function Calling
    

模型原生支持结构化调用外部工具/API，是当前商用 Agent 的基础能力层（OpenAI Function Calling、Claude Tool Use）。

6. ### Multi-Agent（多智能体协作）
    

- **Orchestrator-Subagent**：主控 Agent 分发子任务给专业 Agent
    
- **Debate/Critique**：多 Agent 互相评审，提升输出质量
    
- 代表框架：AutoGen、CrewAI、LangGraph
    

7. ### MCTS / Tree of Thoughts（树状搜索）
    

将推理展开为树结构，通过搜索找到最优路径。适合复杂推理、代码生成等需要探索的任务。

8. ### Memory-Augmented Agent
    

引入多层记忆系统（短期/长期/情景/语义），让 Agent 具备跨会话的持续学习能力。代表：MemGPT。

---

### 范式对比

|   |   |   |
|---|---|---|
|范式|核心思路|适用场景|
|ReAct|推理+行动交替|通用任务|
|Plan-and-Execute|先规划后执行|复杂多步任务|
|ReWOO|批量规划工具调用|效率敏感场景|
|Reflexion|反思迭代|需要自我改进|
|Multi-Agent|分工协作|大型复杂项目|
|ToT/MCTS|树状探索|复杂推理/搜索|

目前工业界最常用的是 **ReAct** **+ Tool Use + Multi-Agent** 的组合方案。
