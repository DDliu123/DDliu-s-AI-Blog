## 一、 开发前的核心准备

1. **定义成功标准**
    
    1. 明确 Agent 的**核心目标、输入内容、输出形式**，确定符合预期的效果指标。
        
2. **准备评测用例集（关键前提）**
    

评测是 Agent 效果量化与迭代的基础，需遵循五步流程：

1. 明确 Agent 目标、输入输出；
    
2. 准备包含「输入数据+人工标注预期结果」的评测集；
    
3. 开发 Agent 初版（保证输出结构即可）；
    
4. 用评测集跑通流程，生成评估报告；
    
5. 分析与预期的差异，迭代优化，重复流程直至达标。
    

6. **快速初始化提示词**
    
    1. 初期无需手工编写，直接用 AI 生成首版系统提示词；
        
    2. 核心目标是**跑通「Agent+评测集+自动化评测」的完整流程**，而非追求提示词完美。
        

## 二、 提示词调优核心方法

1. **用 AI 改进提示词（主流高效方案）**
    
    1. 避免人工反复调整：提示词调优可能需要数十轮，人工成本高、效率低；
        
    2. 使用固定调优模板，需向 AI 提供4类信息：
        
        - 本次输入内容（input）
            
        - Agent 实际输出内容（output）
            
        - 当前系统提示词（system-prompt）
            
        - 具体存在的问题
            
2. **结构性提示词设计技巧**
    
    1. **角色/****目标设定**：不强制赋予角色，明确核心目标即可限定 Agent 工作范围；
        
    2. **思维链（****CoT****）/任务拆分**：将任务拆分为细颗粒度步骤，引导 Agent 逐步完成，提升输出准确度；
        
    3. **采用 XML 格式**：内容按标签分类，支持灵活引用、局部更新，无需严格保证提示词顺序；
        
    4. **添加 Few shot 示例**：在 Agent 达到 60-70 分后使用，解决边缘 case；初期尽量不用，避免过拟合；
        
    5. **精简提示词**：上下文越大，Agent 响应越慢，需在效果与效率间平衡。
        
3. **减少 Few shot** **过拟合**
    
    1. 初期**0 Few shot**，用逻辑表述引导 Agent 理解任务；
        
    2. 后期少量添加示例，避免使用具体名词，防止 Agent 死板输出。
        

## 三、 关键实践经验

1. **精简输入：只给最小必要信息**
    
    1. 避免提供无关内容，防止 Agent 注意力分散；
        
    2. 案例：提速 Agent 仅输入需优化的步骤，响应速度从 60s 降至 6s，输出质量显著提升。
        
2. **拆分小 Agent：单一职责原则**
    
    1. 一个 Agent 只负责一件事，避免多任务耦合导致调优困难；
        
    2. 案例：归因 Agent 拆分为4个小 Agent（分类师、Diff 检查员、分析师、协调者），通过代码串联流程，实现易调优、快响应、影响面可控的效果。
        
3. **做好 Agent 回滚与预期管理**
    
    1. 保管老版本提示词：每次提示词更新都是彻底推翻，负优化时可快速回撤；
        
    2. 接受无法达到 100 分：绝对采纳率达 70% 后易出现负优化，此时可结合 ReAct、RAG 等技术进一步提升，而非执着于提示词调优。
        

## 四、 核心结论

好的 AI Agent 不是一次设计到位的，而是在**「定义标准-评测验证-提示词调优-迭代优化」**的循环中逐步“长”出来的，需重视量化评测与高效的提示词调优方法。
