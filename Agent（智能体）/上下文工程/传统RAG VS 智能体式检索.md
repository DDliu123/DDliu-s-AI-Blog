
#### 1. 传统RAG的“标准答案”与局限性

- **传统RAG的核心逻辑**：以“构建外部结构化系统”为核心，认为AI需要依赖预处理的结构化信息才能高效工作，具体流程可概括为“文档切分→嵌入生成向量→向量数据库存储→提问向量匹配→提取原文生成回答”。
    
- **局限性**：流程复杂（需切分、嵌入、维护向量数据库）、成本高（存储和更新向量数据需持续投入），本质是“用工程师的劳动替代AI的自主判断”，默认AI无法自主筛选和获取信息。
    

#### 2. Claude Code核心开发者Boris提出的“智能体式检索”：反直觉但高效的新方案

这是对传统RAG的彻底颠覆，核心是“相信AI智能，放弃外部索引”，具体包含3个关键要素：

- **核心原则**：不做任何提前索引，把“信息筛选和获取的主动权”交给AI，而非工程师提前预处理。
    
- **两大核心工具/载体**：
    
    - 基础文件操作工具：依赖“grip”（传统程序员常用的简单文本搜索命令），而非复杂的向量匹配算法，满足AI在文件内部的关键词检索需求。
        
    - LMText“地图”文件：非原始文档集合，而是“文档链接+优质内容描述”的目录——核心价值是给AI提供“信息定位线索”，让AI能快速判断“哪些文档可能包含答案”。
        
- **工作流程**：AI接收到问题→读取LMText判断目标文档→调用工具抓取目标文档→（信息不足时）用grip在文档内关键词搜索→基于获取的信息生成回答，完全模拟人类找资料的逻辑。
    

#### 3. 基准测试结果：“智能体式检索”完胜传统方案

测试者斯巴蒂针对“NumberGraph产品文档问答”设计了三种方案对比，结果凸显新方案的优势：

|   |   |   |   |
|---|---|---|---|
|方案类型|具体操作|效果表现|构建与维护成本|
|传统RAG|文档切分→嵌入→向量数据库存储|效果一般|高（需维护向量库）|
|智能体式检索|LMText目录+grip工具|**效果最好**|低（仅需维护目录）|
|暴力方法|直接塞入300万Token完整文档|效果差（大模型处理压力大）|极高（文档传输/处理成本）|

  

关键结论：“智能体式检索”的核心优势并非依赖复杂技术，而是“优质的LMText描述”——只要能让AI准确判断目标文档，就能以极低成本实现比传统RAG更好的效果。

  

  

#### 4. 底层思维转变：从“喂AI”到“赋能AI”

这是新方案最核心的价值，也是顶尖AI开发者的思路迭代：

- **过去的思维**：默认AI“不够聪明”，工程师需承担“信息结构化”的核心工作（如切分文档、生成向量），把“加工好的信息”喂给AI，本质是“外部结构优先于AI智能”。
    
- **现在的思维**：认可大模型的自主判断能力，工程师的工作重心从“构建外部结构”转向“赋能AI自身”——提供基础工具（如grip）、清晰线索（如LMText），让AI自主探索、筛选信息，本质是“AI智能优先于外部结构”。
    

类比理解：如同教孩子学习——传统RAG是“把知识点做成卡片让孩子死记硬背”，智能体式检索是“教孩子用图书馆、查资料的方法，让他自主学习”。

  

  

#### 5. 未来方向：“智能优先于结构”成AI工程重要趋势

视频认为，Boris的方案并非“小众尝试”，而是顶尖开发者基于大模型能力提升的必然选择：随着大模型理解、判断、工具使用能力的增强，“复杂外部结构化系统”的必要性会逐渐降低，未来AI工程的核心会转向“如何更好地释放AI自主能力”——通过优质线索（如清晰的目录、指令）和基础工具，让AI成为能自主解决问题的“智能体”，而非依赖工程师投喂信息的“被动工具”。
