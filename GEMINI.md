# GEMINI Project Context

## Directory Overview

This directory is a comprehensive personal knowledge base (PKM) and product development workspace, managed within an Obsidian vault. The central theme is the research, deconstruction, and strategic planning of **AI Agents**.

The content spans the entire lifecycle of product creation, from deep technical analysis of existing systems to the formulation of a complete business plan and product requirements document for a new venture in the AI Agent space.

## Key Files & Directories

The knowledge base is structured around several key themes:

*   **Product Planning (`Business_Plan.md`, `Product_Requirements_Document_MVP.md`):**
    *   These core documents outline the strategic vision for a **new, easy-to-use AI Agent platform** designed to compete with more complex frameworks. They cover the problem statement, solution, unique value proposition, key metrics, and MVP feature set.

*   **Competitor & Technology Deconstruction (`开源AI项目拆解/`):**
    *   This section contains deep technical analyses of existing AI projects.
    *   `OpenClaw 技术架构与实现原理.md` is a prime example, providing a detailed breakdown of a sophisticated, distributed AI assistant framework. This research forms the basis for the new product strategy.

*   **Agent Development Methodology (`Agent（智能体）/`):**
    *   This is the user's personal wiki on how to build and manage AI Agents.
    *   `Agent 构建与Prompt设计迭代SOP.md` is a crucial file outlining a professional Standard Operating Procedure (SOP) for Agent development. It details a quantifiable, iterative process for prompt engineering, emphasizing a "single responsibility principle" and methodical tuning.

*   **General AI & Product Knowledge (`产品学习笔记/`, `大模型/`, `AI编程/`):**
    *   These directories store foundational knowledge on product management, large language models, and AI-assisted programming, supporting the core focus on Agent development.

## Usage

This directory serves as a "second brain" and strategic workspace. When interacting with this project, assume the following:

*   **Goal-Oriented:** The user is focused on building a real product. Interactions should be practical and aligned with product development goals.
*   **Content-Centric:** The primary artifacts are Markdown files. Common tasks will involve reading, analyzing, summarizing, and creating new documents.
*   **Structured Thinking:** The user appreciates structured, well-organized information, as evidenced by the detailed SOPs and planning documents.
*   **Context is Key:** The user has a deep, existing context on AI Agents. Explanations can be technical and assume a high level of understanding.
